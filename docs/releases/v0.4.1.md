# v0.4.1 Release Notes

Date: 2026-07-05
Changelog: [`0.4.1`](../../CHANGELOG.md)

`v0.4.1` is a compatibility and recovery hardening release for stores created before the v0.4 line settled.

## Highlights

- Legacy source locks are rehashed so `source refresh --check` no longer reports phantom selected-skill drift for older content hashes.
- Clone, fetch, and pull paths preserve user SSH configuration, disable interactive prompts, time out network operations, and clean up failed clone destinations before retry.
- `dalo init` now holds the store lock during repair paths.
- Instruction pack targets preserve symlinks, file modes, and legacy relative lock targets.
- Partial or degraded source scans preserve recorded links instead of removing skills from sources Dalo could not safely inspect.
- Recovery messages for corrupt approvals, leftover adoption backups, and relink races point to more specific next steps.

## Upgrading

Run:

```sh
dalo doctor
dalo status
```

If both commands are clean, run `dalo sync --dry-run` before the first real sync after upgrading.
