# Dalo v0.6.0

## Safer source and state handling

- New catalog sources now start untrusted. Selecting a skill and approving its
  code are separate actions.
- Owner and organization approvals are source-qualified, so a different source
  cannot inherit an approval by declaring the same frontmatter owner.
- Credential-bearing Git URLs are rejected before cloning or persistence, and
  Git diagnostics redact any URL userinfo.
- Sync preserves malformed or newer user locks for recovery. Persisted state,
  lock, catalog, and approval schemas reject unknown fields consistently.
- Instruction-pack writes restore the target file when their matching lock write
  fails. `resolve remove-owned` now requires the generated ID and verifies the
  symlink target before unlinking it.

## Faster catalog work

- Short Git subprocesses no longer wait on a 50 ms polling floor.
- Catalogs hash selected skill content lazily. Unselected offers retain metadata
  without paying the full content-hash cost.

## Distribution

- GitHub release archives are keylessly signed with Sigstore/Cosign. The shell
  installer verifies the bundle and release workflow identity before extracting.
- `npx dalo` and `npm install --global dalo` are supported through the new
  cross-platform wrapper. It verifies the release checksum before caching and
  executing the native binary.
- Install docs include `mise use -g ubi:sebastian-software/dalo`.

## Upgrade notes

- Re-approve owner records as `<source-id>:<owner>`.
- Install Cosign before using the shell installer so it can verify release
  provenance.
- Existing catalog selections stay selected, but require an explicit approval
  before they materialize from an untrusted catalog source.
