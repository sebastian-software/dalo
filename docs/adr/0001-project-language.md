# ADR 0001: Project Language

Status: Accepted  
Date: 2026-06-23  

## Context

Skillmgr is intended to be a team-oriented tool with public-facing documentation, CLI copy, RFCs, issues, pull requests, and code comments. Mixed project language would make review, contribution, and generated-agent work less predictable.

## Decision

English is the project language.

Product documents, RFCs, ADRs, CLI copy, code comments, commit messages, issues, and pull requests should be written in English unless they quote or reference external material in another language.

## Consequences

- User-facing and contributor-facing text stays consistent.
- RFCs and ADRs can be reviewed by a broader contributor base.
- Non-English discussion can still happen informally, but committed project artifacts should use English.
